js原生中 onclick添加事件处理函数是在全局环境下执行，污染了全局环境， 且给很多dom元素添加onclick事件，影响网页的性能， 同时如果动态的从dom树种删除了该元素，还要手动注销事件处理器，不然就可能造成内存泄露 React里的事件 挂载的函数都控制在组件范围内，不会污染全局空间 react中的事件采取了事件委托的方式，挂载最顶层DOM节点，也意味着 React 在更新 DOM 时不需要跟踪事件监听器。提升页面性能 但是React里面的event并不是原生，他是合成的事件对象 加上React控制了组件的生命周期，在unmount的时候自然能够清楚相关的所有事件处理函数，内存泄露不再是一个问题
